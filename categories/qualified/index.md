# Qualified Terms {=mdp:categories#qualified .mdp:Category label}

The Qualified Terms category is the result of applying the Qualification Pattern [LD-Patterns-QR] to the simple (unqualified) relations available in the Starting Point and Expanded categories. The terms in this category are for users who wish to provide further details about the provenance-related influence among Entities, Activities, and Agents.

The Qualification Pattern restates an unqualified influence relation by using an intermediate class that represents the influence between two resources. This new instance, in turn, can be annotated with additional descriptions of the influence that one resource had upon another. The following two tables list the influence relations that can be qualified using the Qualification Pattern, along with the properties used to qualify them. For example, the second row of the first table indicates that to elaborate how an prov:Activity prov:used a particular prov:Entity, one creates an instance of prov:Usage that indicates the influencing entity with the prov:entity property. Meanwhile, the influenced prov:Activity indicates the prov:Usage with the property prov:qualifiedUsage. The resulting structure that qualifies the an Activity's usage of an Entity is illustrated in Figure 4a below.

## Starting Point Qualification Patterns

Seven Starting Point relations can be further described using the Qualification Pattern:

### Entity Generation Qualification {=mdp:qual:entity-generation .mdp:QualificationPattern}

When an [Entity] {+prov:Entity ?mdp:influencedClass} is generated by an [Activity] {+prov:Activity ?mdp:influencingClass}, you can qualify this relationship using [wasGeneratedBy] {+prov:wasGeneratedBy ?mdp:unqualifiedInfluence}. The qualification uses [qualifiedGeneration] {+prov:qualifiedGeneration ?mdp:qualificationProperty} to reference a [Generation] {+prov:Generation ?mdp:qualifiedInfluence} instance, which then cites the generating [Activity] {+prov:activity ?mdp:influencerProperty}.

### Entity Derivation Qualification {=mdp:qual:entity-derivation .mdp:QualificationPattern}

When one [Entity] {+prov:Entity ?mdp:influencedClass} is derived from another [Entity] {+prov:Entity ?mdp:influencingClass}, you can qualify this relationship using [wasDerivedFrom] {+prov:wasDerivedFrom ?mdp:unqualifiedInfluence}. The qualification uses [qualifiedDerivation] {+prov:qualifiedDerivation ?mdp:qualificationProperty} to reference a [Derivation] {+prov:Derivation ?mdp:qualifiedInfluence} instance, which then cites the source [entity] {+prov:entity ?mdp:influencerProperty}.

### Entity Attribution Qualification {=mdp:qual:entity-attribution .mdp:QualificationPattern}

When an [Entity] {+prov:Entity ?mdp:influencedClass} is attributed to an [Agent] {+prov:Agent ?mdp:influencingClass}, you can qualify this relationship using [wasAttributedTo] {+prov:wasAttributedTo ?mdp:unqualifiedInfluence}. The qualification uses [qualifiedAttribution] {+prov:qualifiedAttribution ?mdp:qualificationProperty} to reference an [Attribution] {+prov:Attribution ?mdp:qualifiedInfluence} instance, which then cites the responsible [agent] {+prov:agent ?mdp:influencerProperty}.

### Activity Usage Qualification {=mdp:qual:activity-usage .mdp:QualificationPattern}

When an [Activity] {+prov:Activity ?mdp:influencedClass} [used] {+prov:used ?mdp:unqualifiedInfluence} an [Entity] {+prov:Entity ?mdp:influencingClass}, you can qualify this relationship. The qualification uses [qualifiedUsage] {+prov:qualifiedUsage ?mdp:qualificationProperty} to reference a [Usage] {+prov:Usage ?mdp:qualifiedInfluence} instance, which then cites the used [entity] {+prov:entity ?mdp:influencerProperty}.

### Activity Communication Qualification {=mdp:qual:activity-communication .mdp:QualificationPattern}

When one [Activity] {+prov:Activity ?mdp:influencedClass} [wasInformedBy] {+prov:wasInformedBy ?mdp:unqualifiedInfluence} another [Activity] {+prov:Activity ?mdp:influencingClass}, you can qualify this relationship. The qualification uses [qualifiedCommunication] {+prov:qualifiedCommunication ?mdp:qualificationProperty} to reference a [Communication] {+prov:Communication ?mdp:qualifiedInfluence} instance, which then cites the informing [activity] {+prov:activity ?mdp:influencerProperty}.

### Activity Association Qualification {=mdp:qual:activity-association .mdp:QualificationPattern}

When an [Activity] {+prov:Activity ?mdp:influencedClass} [wasAssociatedWith] {+prov:wasAssociatedWith ?mdp:unqualifiedInfluence} an [Agent] {+prov:Agent ?mdp:influencingClass}, you can qualify this relationship. The qualification uses [qualifiedAssociation] {+prov:qualifiedAssociation ?mdp:qualificationProperty} to reference an [Association] {+prov:Association ?mdp:qualifiedInfluence} instance, which then cites the associated [agent] {+prov:agent ?mdp:influencerProperty}.

### Agent Delegation Qualification {=mdp:qual:agent-delegation .mdp:QualificationPattern}

When one [Agent] {+prov:Agent ?mdp:influencedClass} [actedOnBehalfOf] {+prov:actedOnBehalfOf ?mdp:unqualifiedInfluence} another [Agent] {+prov:Agent ?mdp:influencingClass}, you can qualify this relationship. The qualification uses [qualifiedDelegation] {+prov:qualifiedDelegation ?mdp:qualificationProperty} to reference a [Delegation] {+prov:Delegation ?mdp:qualifiedInfluence} instance, which then cites the delegating [agent] {+prov:agent ?mdp:influencerProperty}.

When an [Activity] {+prov:Activity ?mdp:influencedClass} [wasEndedBy] {+prov:wasEndedBy ?mdp:unqualifiedInfluence} an [Entity] {+prov:Entity ?mdp:influencingClass}, you can qualify this relationship. All influence classes (e.g. prov:Association, prov:Usage) are extensions of prov:Influence and either prov:EntityInfluence, prov:ActivityInfluence, or prov:AgentInfluence, which determine the property used to cite the influencing resource (either prov:entity, prov:activity, or prov:agent, respectively). Because prov:Influence is a broad relation, its most specific subclasses (e.g. prov:Communication, prov:Delegation, prov:End, prov:Revision, etc.) should be used when applicable.

As can be seen in this example, qualifying an influence relation provides a second form (e.g. :e1 prov:qualifiedGeneration :e1Gen) to express an equivalent influence relation (e.g. :e1 prov:wasGeneratedBy :a1). It is correct and acceptable for an implementer to use either qualified or unqualified forms as they choose (or both), and a consuming application should be prepared to recognize either form. Consuming applications should recognize both qualified and unqualified forms, and treat the qualified form as implying the unqualified form. Because the qualification form is more verbose, the unqualified form should be favored in cases where additional properties are not provided. When the qualified form is expressed, including the equivalent unqualified form can facilitate PROV-O consumption, and is thus encouraged.

In addition to the previous two tables, Figure 4 illustrates the classes and properties needed to apply the qualification pattern to ten of the fourteen qualifiable influence relations. For example, while prov:qualifiedUsage, prov:Usage, and prov:entity are used to qualify prov:used relations, prov:qualifiedAssociation, prov:Association, and prov:agent are used to qualify prov:wasAssociatedWith relations. This pattern applies to the twelve other influence relations that can be qualified.

In subfigure a the prov:qualifiedUsage property parallels the prov:used property and references an instance of prov:Usage, which in turn provides attributes of the prov:used relation between the Activity and Entity. The prov:entity property is used to cite the Entity that was used by the Activity. In this case, the time that the Activity used the Entity is provided using the prov:atTime property and a literal xsd:dateTime value. The prov:atTime property can be used to describe any prov:InstantaneousEvent (including prov:Start, prov:Generation, prov:Usage, prov:Invalidation, and prov:End).

Similarly in subfigure j, the prov:qualifiedAssociation property parallels the prov:wasAssociatedWith property and references an instance of prov:Association, which in turn provides attributes of the prov:wasAssociatedWith relation between the Activity and Agent. The prov:agent property is used to cite the Agent that influenced the Activity. In this case, the plan of actions and steps that the Agent used to achieve its goals is provided using the prov:hadPlan property and an instance of prov:Plan. Further, the prov:hadRole property and prov:Role class can be used to describe the function that the agent served with respect to the Activity. Both prov:Plan and prov:Role are left to be extended by applications.
