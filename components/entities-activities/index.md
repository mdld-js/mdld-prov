## Summary

The PROV-O Entities and Activities component provides the foundational building blocks of provenance, modeling the "what" and "when" of provenance through entities, activities, and their temporal relationships. The component is organized into clear flows that separate concerns while maintaining consistent patterns.

**Flow Architecture:**
Each flow follows the provenance pattern of binary relations, qualified forms, and influence classes, with temporal properties providing cross-cutting time annotations.

**Core Flows:**

**1. Entities Flow** - Fundamental provenance objects
- **Entity**: Physical, digital, or conceptual things with fixed aspects
- **value**: Direct representation of entity characteristics
- **Location**: Spatial context for entities and activities
- **InstantaneousEvent**: Events marking world transitions
- Establishes the "what" of provenance statements

**2. Entity Lifecycle Flow** - Entity state transitions
- `wasGeneratedBy` → `qualifiedGeneration` → `Generation`
- `wasInvalidatedBy` → `qualifiedInvalidation` → `Invalidation`
- **Temporal Properties**: `generatedAtTime`, `invalidatedAtTime`
- Models creation, existence, and destruction of entities
- Example: Entity generated by activity, later invalidated

**3. Activity Lifecycle Flow** - Activity state management
- `wasStartedBy` → `qualifiedStart` → `Start`
- `wasEndedBy` → `qualifiedEnd` → `End`
- **Temporal Properties**: `startedAtTime`, `endedAtTime`
- Models initiation and termination of activities
- Example: Activity started by trigger entity, ended by terminator entity

**4. Usage of Entities by Activities Flow** - Core interaction pattern
- `used` → `qualifiedUsage` → `Usage`
- Models consumption, processing, transformation of entities by activities
- Fundamental provenance relationship (activities act upon entities)
- Example: Baking activity uses spoon, egg, oven

**5. Inter-Activity Communication Flow** - Activity coordination
- `wasInformedBy` → `qualifiedCommunication` → `Communication`
- Models exchange of entities between activities
- Supports activity dependency and information flow
- Example: Activity A2 informed by Activity A1 via shared entity

**6. Temporal Properties Flow** - Cross-cutting time framework
- `atTime`: For instantaneous events (generation, usage, start, end, invalidation)
- **Activity Timing**: `startedAtTime`, `endedAtTime`
- **Entity Timing**: `generatedAtTime`, `invalidatedAtTime`
- Provides temporal context for all provenance elements
- Supports property chains for qualified temporal access

**Key Relationships:**
```
Entity (disjoint from Activity)
├── Entity Lifecycle
│   ├── Generation (creates entities)
│   └── Invalidation (destroys entities)
└── Activity Lifecycle
    ├── Start (initiates activities)
    └── End (terminates activities)

Activity-Entity Interactions
├── Usage (activities use entities)
└── Communication (activities coordinate via entities)

Temporal Framework
├── atTime (instantaneous events)
├── Entity Timing (generatedAtTime, invalidatedAtTime)
└── Activity Timing (startedAtTime, endedAtTime)
```

**Benefits of Flow Organization:**
- Clear separation of entity vs activity concerns
- Consistent binary→qualified→influence patterns
- Explicit temporal modeling across all flows
- Logical lifecycle progression modeling
- Cross-cutting spatial and temporal context
- Foundation for all other PROV-O components
